# 한국인 개발자 최적화 가이드 (Korean Dev Standard)

이 규칙은 이 프로젝트를 담당하는 한국인 개발자 전용이며, IDE 설정이나 개인 설정 파일로만 로드됩니다.

## 1. 지성 및 소통 언어 (Communication)
- **기본 언어**: 모든 질문, 답변, 설명, 오류 분석은 반드시 **한국어**로 수행한다.
- **존칭 사용**: 전문적인 소프트웨어 엔지니어로서 한국어 경어체를 사용하여 정중하게 답변한다.

## 2. 프로젝트 기술 문맥 (Context)
- **구성**: React 기반의 프론트엔드 프로젝트로 백엔드와 REST API 통신 구조를 가진다.
- **컨벤션**: 기존 코드의 React Hook 사용 방식 및 컴포넌트 구조를 일관성 있게 유지한다.

## 3. 작업 프로세스 (Work Process)
- **선제적 제안**: UI/UX 변경이나 새로운 컴포넌트 추가 전, 구조에 대해 한국어로 먼저 제안한다.
- **에러 분석**: 브라우저 콘솔 에러나 API 통신 오류 발생 시 원인을 한국어로 상세 설명한다.
